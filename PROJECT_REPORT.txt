================================================================================
                                EXECUTIVE SUMMARY
================================================================================

Manobala is a comprehensive mental health support platform designed to provide 
a safe, supportive environment for individuals seeking mental health resources, 
professional guidance, and community support. The platform combines modern web 
technologies to deliver a full-stack solution addressing critical mental health 
needs in today's society.

================================================================================
                              PROJECT OVERVIEW
================================================================================

MISSION:
To create an accessible, user-friendly platform that bridges the gap between 
individuals seeking mental health support and qualified professionals while 
fostering a supportive community environment.

KEY OBJECTIVES:
1. Provide easy access to mental health resources
2. Connect users with qualified mental health experts
3. Create safe spaces for peer support and community discussions
4. Offer educational content and coping strategies
5. Implement secure user authentication and data protection
6. Enable real-time communication and support

================================================================================
                              DATABASE DESIGN
================================================================================

USER SCHEMA:
- _id: ObjectId (Primary Key)
- name: String (Required)
- email: String (Unique, Required)
- password: String (Hashed)
- isVerified: Boolean (Email verification status)
- verificationToken: String
- resetPasswordToken: String
- resetPasswordExpires: Date
- role: String (user/expert/admin)
- profileImage: String (URL)
- bio: String
- expertise: Array (For experts)
- isExpertApproved: Boolean
- expertApplicationId: ObjectId
- createdAt: Date
- updatedAt: Date

FORUM SCHEMA:
- _id: ObjectId (Primary Key)
- name: String (Required)
- description: String (Required)
- type: String (Unique identifier)
- category: String (Forum category)
- icon: String (Emoji representation)
- color: String (UI color scheme)
- memberCount: Number (Active members)
- messages: Array of ObjectId (Message references)
- moderators: Array of ObjectId (User references)
- createdAt: Date
- isActive: Boolean

MESSAGE SCHEMA:
- _id: ObjectId (Primary Key)
- content: String (Required)
- author: ObjectId (User reference)
- roomId: ObjectId (Forum reference)
- isAnonymous: Boolean
- reactions: Array of reaction objects
- replies: Array of reply objects
- isEdited: Boolean
- editedAt: Date
- createdAt: Date

EXPERT APPLICATION SCHEMA:
- _id: ObjectId (Primary Key)
- userId: ObjectId (User reference)
- qualifications: String
- experience: String
- specializations: Array
- licenseNumber: String
- documents: Array of file objects
- status: String (pending/approved/rejected)
- reviewedBy: ObjectId (Admin reference)
- reviewNotes: String
- submittedAt: Date
- reviewedAt: Date

================================================================================
                               API ENDPOINTS
================================================================================

AUTHENTICATION ROUTES:
POST /api/auth/register       - User registration
POST /api/auth/login          - User login
POST /api/auth/logout         - User logout
GET  /api/auth/is-auth        - Authentication check
POST /api/auth/verify-email   - Email verification
POST /api/auth/forgot-password- Password reset request
POST /api/auth/reset-password - Password reset confirmation

USER ROUTES:
GET  /api/user/profile        - Get user profile
PUT  /api/user/profile        - Update profile
POST /api/user/apply-expert   - Expert application
GET  /api/user/expert-status  - Application status

FORUM ROUTES:
GET  /api/forum/rooms         - Get all forum rooms
GET  /api/forum/rooms/:id/messages - Get room messages
POST /api/forum/rooms/:id/messages - Post new message
POST /api/forum/messages/:id/reactions - Add reaction
POST /api/forum/messages/:id/replies - Add reply
POST /api/forum/rooms/:id/join - Join room

BLOG ROUTES:
GET  /api/blog/all           - Get all blogs
GET  /api/blog/user/:id      - Get user blogs
POST /api/blog/create        - Create new blog
PUT  /api/blog/:id          - Update blog
DELETE /api/blog/:id        - Delete blog

ADMIN ROUTES:
GET  /api/admin/users        - Get all users
GET  /api/admin/expert-applications - Get applications
PUT  /api/admin/expert-applications/:id - Review application
DELETE /api/admin/user/:id   - Delete user

================================================================================
                              CHALLENGES & SOLUTIONS
================================================================================

CHALLENGES ENCOUNTERED:

1. DATABASE CONNECTIVITY:
   Problem: Connection timeout issues during forum initialization
   Solution: Implemented proper connection configuration with timeout settings
            and error handling

2. AUTHENTICATION FLOW:
   Problem: JWT token management and user session persistence
   Solution: HTTP-only cookies with proper expiration and refresh mechanisms

3. REAL-TIME MESSAGING:
   Problem: Socket.IO connection management and room handling
   Solution: Proper event listeners and connection lifecycle management

4. DATE FORMATTING:
   Problem: Invalid date values causing frontend errors
   Solution: Comprehensive error handling and date validation in components

5. USER ROLE MANAGEMENT:
   Problem: Complex authorization logic for different user types
   Solution: Middleware-based role checking and route protection

6. FILE UPLOAD HANDLING:
   Problem: Expert document verification and storage
   Solution: Multer integration with proper validation and error handling
